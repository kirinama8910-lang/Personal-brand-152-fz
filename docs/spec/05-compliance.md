## 152-ФЗ чеклист — где что разместить

| Требование | Где на лендинге | Статус |
|-----------|-----------------|--------|
| Cookie-banner с равными кнопками «Принять» / «Отклонить» | Появляется на первом заходе, до запуска любых не-essential скриптов | реализовать в верстке |
| Все чекбоксы согласия — пустые по умолчанию | Cookie-banner + fallback-форма квиза | реализовать |
| Раздельные чекбоксы на каждую цель обработки | Согласие на ОПД, согласие на маркетинг — отдельно | реализовать (с 01.09.2025 обязательно) |
| Кнопка submit заблокирована, пока не отмечен обязательный чекбокс | Fallback-форма квиза | реализовать |
| Ссылка на Политику конфиденциальности — рядом с каждым чекбоксом | Cookie-banner + форма | реализовать |
| Шрифты self-host, серверы в РФ | `/assets/fonts/` + размещение на Timeweb (РФ) | см. раздел 7 |
| Никаких foreign CDN | весь сайт | см. раздел 7 |
| Никакого Google Maps, reCAPTCHA, Facebook Pixel | весь сайт | проверить перед запуском |
| Метрика — динамическая инжекция после согласия, не статический тег в `<head>` | Реализовать в `analytics.js` | реализовать |
| Ссылка «Управление согласиями» в footer | Секция 12 — пункт «Управление согласиями» → `/consent` | реализовать |
| Privacy Policy / Согласие / Cookie Policy — доступны | `/privacy-policy.html`, `/consent.html`, `/cookie-policy.html` | **существуют, см. раздел 9** |

Полный чеклист требований — в [.claude/skills/frontend-design/SKILL.md](.claude/skills/frontend-design/SKILL.md), раздел «152-FZ Design Compliance».

## Статус документов

В корне репозитория уже существуют три юридических документа:

| Файл | Что внутри | Статус |
|------|-----------|--------|
| [privacy-policy.html](privacy-policy.html) | Политика конфиденциальности | готов, без обнаруженных плейсхолдеров |
| [consent.html](consent.html) | Согласие на обработку персональных данных | **содержит маркер `[ЗАПОЛНИТЬ]`** — заполнить вручную перед запуском |
| [cookie-policy.html](cookie-policy.html) | Политика использования cookie | готов, без обнаруженных плейсхолдеров |

### Что заполнить вручную перед запуском

1. Открыть [consent.html](consent.html), найти все вхождения `[ЗАПОЛНИТЬ]`, подставить реальные данные оператора (ФИО ИП, ИНН, email, адрес).
2. Перепроверить три документа на актуальность дат и реквизитов оператора.
3. Зафиксировать в верстке лендинга **одинаковые ссылки** на эти три файла из:
   - cookie-banner (рядом с чекбоксами);
   - fallback-формы квиза (рядом с чекбоксами);
   - footer (секция 12);
   - страницы благодарности после квиза.

### Чего НЕ делать

- **Не править плейсхолдеры в юр. документах при работе над лендингом.** Эти документы готовятся отдельно, копирайт лендинга в них не вторгается.
- Не дублировать тексты политики/согласия в собственный HTML лендинга — только ссылаться.
