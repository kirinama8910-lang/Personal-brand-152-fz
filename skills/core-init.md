# 152-ФЗ Compliance Skill для Claude Code
**Версия 2.0 | Актуально: апрель 2026 г.**

Помогает создавать и дорабатывать сайты и мобильные приложения, изначально соответствующие 152-ФЗ. Принцип: **не исправлять нарушения постфактум, а не допускать их с самого начала.**

Поддерживаемые типы проектов: `landing` · `corporate` · `ecommerce` · `saas` · `mobile`

---

## ШАГ 0: ИНИЦИАЛИЗАЦИЯ (выполнять первым при каждом старте)

При активации скилла задай разработчику **5 вопросов подряд одним сообщением**. Заполни переменные из ответов — они используются во всех последующих фазах.

```
Чтобы сгенерировать корректные документы и код, ответь на 5 вопросов:

1. Тип проекта:
   a) Лендинг / простой сайт-визитка
   b) Корпоративный сайт
   c) Интернет-магазин
   d) SaaS / веб-приложение с авторизацией
   e) Мобильное приложение (iOS / Android / React Native)

2. Технологический стек (выбери один или несколько):
   HTML + Vanilla JS / React / Vue / Next.js / Nuxt / другой

3. Данные оператора (можно скопировать из документов):
   - Полное наименование (ООО, ИП, АО...):
   - ИНН:
   - ОГРН/ОГРНИП:
   - Юридический адрес:
   - Email ответственного за ПДн:

4. Хостинг-провайдер и где физически стоят серверы:
   (например: Selectel, серверы в Москве)

5. Какие данные собирает сайт? (отметь все подходящие)
   [ ] Имя, телефон, email (контактная форма)
   [ ] Адрес доставки
   [ ] Данные банковских карт
   [ ] Учётные записи (логин/пароль)
   [ ] Данные о здоровье / биометрия
   [ ] Файлы cookie / аналитика поведения
```

После получения ответов:
1. Подставь значения в переменные `{{...}}` по всему документу
2. Определи тип проекта по матрице ниже
3. Выполни только **релевантные фазы** для этого типа
