# Квиз-диагностика для лендинга фрилансера: дизайн

*Дата: 2026-04-24. На основе [segmentation_hypotheses.md](../../../segmentation_hypotheses.md).*

---

## 1. Контекст и цели

**Продукт:** услуги фрилансера-одиночки — автоматизация бизнеса (n8n), Telegram-боты, сайты/лендинги на вайбкодинге.

**Проблема:** лендинга пока нет. Нужен квиз — главный механизм конверсии анонимного посетителя в тёплого лида.

**Цель квиза:** превратить анонимного посетителя в лида, который:
1. Оставил Telegram-контакт.
2. Уже понимает свою проблему в деньгах (видит оценочный диапазон потерь на финальном экране).
3. Ждёт личного сообщения — не звонка, не email-рассылки.

**Целевые ниши (из сегментации, БЛОК 1):**
- **Эксперты-одиночки** — психологи, коучи, нутрициологи, репетиторы, методологи. Приоритет 1 в плане запуска.
- **Салоны красоты / студии** 2–5 мастеров. Приоритет 2.

**Не-ЦА (квиз направляет в мини-форму, не в полный сценарий):**
- Клиники (стоматологии, ветеринарии, косметология) — обрабатываются отдельным треком по ABCDX.
- Другие отрасли.

## 2. Архитектура — скелет из 7 экранов

| # | Экран | Для кого | Что делает |
|---|-------|----------|-----------|
| 0 | Hero + CTA | оба | Первичный хук, ожидание, запуск |
| 1 | Q1 — Боль | оба | Триггер-вход + предварительная классификация |
| 2 | Q2 — Формат бизнеса | оба | Окончательный классификатор Эксперт/Салон + выход для не-ЦА |
| 3 | Q3 — Инструменты | ветвится | Сбор A-сигналов, контекст для бэка |
| 4 | Q4 — Масштаб (числовой ввод) | ветвится | Данные для калькулятора потерь |
| 5 | Q5 — Срочность | оба | A/B/C-скоринг |
| 6 | Финал | оба | Диапазон потерь + TG deep link + fallback-форма |

**Принципы UX:**

- **Один экран = один вопрос.** Прогресс-бар sticky-top, нумерация 1/5…5/5 (Q0 hero и финал не считаются).
- **Кнопка «← назад» везде**, кроме hero и финала.
- **Ветвление после Q2.** Q1 — ориентировочная классификация, Q2 — окончательная. Вопросы Q3 и Q4 — специфичны под ветку (Эксперт/Салон), Q5 — универсальный.
- **State хранится в `sessionStorage` до финала**, отправка payload — один раз.
- **Никакого промежуточного захвата контакта.** Только финал или выход для не-ЦА.
- **Fail-safe.** Закрыл вкладку на Q3 — ответы уходят в Яндекс.Метрику как события, но не в бэк (приватность).
- **Длина: 5 вопросов.** Меньше — не хватает сигналов для скоринга и калькулятора; больше — завершаемость падает на 15–25% на B2B-аудитории.
- **Mobile-first.** Основной трафик — с мобильных (TG-чаты, Я.Карты).

## 3. Вопросы квиза — текст и логика

### Hero (экран 0)

- **H1:** «Посчитайте, сколько денег ваш бизнес теряет каждый месяц»
- **Подзаголовок:** «5 вопросов, 2 минуты. На выходе — диапазон потерь в рублях и план возврата в Telegram»
- **CTA-кнопка:** «Начать — 2 минуты»
- **Микро-футер под кнопкой:** «Без регистрации. Контакт оставляете только если сами захотите в конце»

### Q1 — Боль (универсальный)

**Текст:** «Что сильнее всего мешает зарабатывать больше прямо сейчас?»
**Тип:** single choice, обязательный.

| # | Вариант | Класс. |
|---|---------|--------|
| 1 | 📞 Записи теряются после 20:00 — клиенты пишут, когда некому ответить, и уходят в соседнее место | → Салон (strong) |
| 2 | 📉 Нет стабильного потока новых клиентов: Instagram больше не работает, сарафан не тянет | → Эксперт (strong) |
| 3 | 🔁 Рутина съедает день: каждое типовое сообщение — вручную, до ночи в мессенджерах | → ambiguous |
| 4 | 💸 Клиенты приходят один раз и пропадают — возвратов почти нет | → ambiguous |
| 5 | 🎯 Сайт/канал есть, но заявок с него нет — деньги на рекламу улетают в пустоту | → Эксперт (lean) |

Хранится как `q1_pain` + `niche_guess`. Окончательная классификация — на Q2.

### Q2 — Формат бизнеса (универсальный, окончательный классификатор)

**Текст:** «Коротко про ваш формат работы:»
**Тип:** single choice, обязательный.

| # | Вариант | Класс. |
|---|---------|--------|
| 1 | 👤 Я работаю самостоятельно — продаю своё время (психолог, коуч, нутрициолог, репетитор, методолог) | → ветка **Эксперт** |
| 2 | ✂️ У меня студия / салон: 2–5 мастеров, принимаем клиентов в одном месте | → ветка **Салон** |
| 3 | 🏥 Клиника с врачами (стоматология, ветеринария, косметология с медлицензией) | → **выход `out_clinic`** |
| 4 | 🛒 Другое (производство, розница, онлайн-школа…) | → **выход `out_other`** |

### Q3 — Инструменты (ветвится; multi-select до 3 вариантов)

**Ветка Эксперт.** Текст: «Где сейчас ведёте практику и продвигаетесь?»

| Вариант | A/B-сигнал |
|---------|------------|
| Telegram-канал (мой, веду регулярно) | A |
| Instagram через VPN | нейтрально |
| Ясно / Meta / Alter / Профи.ру (агрегаторы) | A |
| Свой сайт / Tilda / Readymag | нейтрально |
| Авито Услуги | нейтрально |
| Только сарафан, никакой площадки | B |

**Ветка Салон.** Текст: «Что используете для записи клиентов и общения с ними?»

| Вариант | A/B-сигнал |
|---------|------------|
| YCLIENTS | A |
| DIKIDI | A |
| Aloxo / Sonline / другой online-сервис | A |
| Telegram-канал с записью через @username | нейтрально |
| Встроенный бот от YCLIENTS/DIKIDI | A+сигнал проблемы (не интегрируется с календарём) |
| Excel / тетрадь / админ вручную | C |

### Q4 — Масштаб (ветвится; два микро-вопроса на одном экране)

**Ветка Эксперт.**

- **Q4a:** «Средний чек одной консультации/сессии?» → До 2 000 / 2–4 000 / 4–7 000 / 7–15 000 / свыше 15 000 ₽
- **Q4b:** «Сколько консультаций в среднем проводите в месяц?» → До 20 / 20–40 / 40–80 / 80+

**Ветка Салон.**

- **Q4a:** «Средний чек одного клиента за визит?» → До 1 500 / 1 500–3 000 / 3 000–6 000 / 6 000–12 000 / свыше 12 000 ₽
- **Q4b:** «Сколько записей/обращений в неделю пропускаете, когда некому ответить (вечером, в выходные, админ не на месте)?» → 1–3 / 4–7 / 8–15 / 15+

**Mid-значения интервалов** (используются калькулятором):

| Q4 ввод | mid |
|---------|-----|
| До 2 000 / До 1 500 | 1000 / 750 |
| 2–4 000 / 1,5–3 000 | 3000 / 2250 |
| 4–7 000 / 3–6 000 | 5500 / 4500 |
| 7–15 000 / 6–12 000 | 11000 / 9000 |
| >15 000 / >12 000 | 18000 / 15000 |
| До 20 / 1–3 | 10 / 2 |
| 20–40 / 4–7 | 30 / 5.5 |
| 40–80 / 8–15 | 60 / 11.5 |
| 80+ / 15+ | 90 / 17 |

### Q5 — Срочность (универсальный, для A/B/C-скоринга)

**Текст:** «Если найдётся решение, которое реально возвращает эти деньги — когда хотите запуститься?»
**Тип:** single choice.

| Вариант | Скоринг |
|---------|---------|
| 🔥 Нужно прямо сейчас — болит уже долго | A |
| 📅 В ближайший месяц | A/B |
| 🗓 В течение квартала | B |
| 👀 Просто смотрю варианты, ничего срочного | C/D |

## 4. A/B/C/X-скоринг (невидимый для пользователя)

Скоринг вычисляется на фронте и кладётся в payload. Логика:

**Маппинг `scale_signal` (из Q4b):**

| Q4b ответ (эксперт / салон) | `scale_signal` |
|----------------------------|----------------|
| «До 20» / «1–3» | `low` |
| «20–40» / «4–7» | `mid` |
| «40–80» / «8–15» | `mid` |
| «80+» / «15+» | `high` |

**Правила для тега:**

- **A (звонить сегодня):** `urgency = "now"` **И** Q3 содержит A-сигнал (Ясно/Meta/TG-канал / YCLIENTS/DIKIDI/Aloxo) **И** `scale_signal ∈ {mid, high}`.
- **B (греть контентом):** `urgency ∈ {now, month}` И один из A-сигналов отсутствует.
- **C (контент-воронка, не тратить время):** `urgency = "looking"` ИЛИ Q3 = «только сарафан» ИЛИ `scale_signal = "low"`.
- **X (премиум-трек):** Q2 = салон И Q4a = «>12 000 ₽» И Q4b = «15+ пропущенных» → косвенный X-сегмент сети.

В уведомлении вам показывается тег + 3 сигнала:
- Срочность
- Масштаб (из Q4b)
- Инструменты (A-признак или C-признак из Q3)

Этого достаточно, чтобы за 10 секунд решить — писать лично сегодня или уводить в контент.

## 5. Калькулятор потерь

### 5.1. Ветка Эксперт — три сценария

| Сценарий | Условие | Формула |
|----------|---------|---------|
| **A. Комиссия агрегаторов** | Q3 содержит Ясно / Meta / Alter / Профи.ру | `loss_min = sessions × price × 0.30` <br> `loss_max = sessions × price × 0.50` |
| **B. Недобор потока** | Q3 не содержит агрегаторов, Q1 in {2, 5} | `target = 40` (медиана по сегментации) <br> `gap = max(target - sessions, 0)` <br> `loss_min = gap × price × 0.5` <br> `loss_max = gap × price × k_max`, где `k_max = 0.7` если Q3 = «только сарафан», иначе `1.0` <br> *Если `gap = 0` — падаем на сценарий C.* |
| **C. Рутина / возвраты** | Q1 in {3, 4}, иначе остаточный случай | `loss_min = sessions × price × 0.15` <br> `loss_max = sessions × price × 0.30` |

### 5.2. Ветка Салон — одна формула

```
missed_per_week = Q4b.mid            // 2 / 5.5 / 11.5 / 17
avg_check       = Q4a.mid            // 750 / 2250 / 4500 / 9000 / 15000
loss_min = missed_per_week × 4.3 × avg_check × 0.5
loss_max = missed_per_week × 4.3 × avg_check × 0.8
```

0.5–0.8 — доля реально возвращаемых пропущенных записей (консервативно / оптимистично).

### 5.3. Санитарные правила

- Округление до тысяч: `Math.round(value / 1000) * 1000`.
- Минимум `loss_max < 10000`: цифру не показываем. Вместо неё — «Потери в вашем случае небольшие, но в деньгах всё равно есть куда расти. Покажу где именно.» + та же TG-кнопка.
- Максимум: `loss_max` ограничиваем 500 000 ₽ (cap — чтобы не выглядело кликбейтом).
- Формат: «47 000 – 75 000 ₽/мес», пробелы в разрядах.

## 6. Финальный экран

### 6.1. Структура

- **Заголовок:** «По вашим ответам, вы теряете ориентировочно»
- **Крупная цифра:** «47 000 – 75 000 ₽ в месяц»
- **Строка-причина** (по сценарию):
  - Эксперт-A: «Агрегаторы забирают 30–50% с каждой сессии»
  - Эксперт-B: «Свободные слоты в расписании остаются пустыми — это и есть потери»
  - Эксперт-C: «Без автоматизации напоминаний и возвратов теряется 15–30% выручки»
  - Салон: «Это записи, которые уходят, когда некому ответить после 20:00»
- **Горизонтальная прогресс-полоса** с двумя метками: «теряете сейчас / могли бы зарабатывать».

### 6.2. Главный путь — TG deep link

Большая кнопка:
```
[ Получить точный расчёт и план за 15 минут в Telegram ]
  → t.me/YourBot?start=sess_xxx
```

Под кнопкой мелким: «Нажимая кнопку, вы соглашаетесь с [Политикой конфиденциальности]».

### 6.3. Fallback-форма (под разделителем «или»)

```
  Ваш @username в Telegram или MAX
  ┌────────────────────────────────┐
  │ @username                      │
  └────────────────────────────────┘

  ☐ Я даю [Согласие на обработку персональных данных]
    и принимаю [Политику конфиденциальности]

         [    Отправить    ]
```

- Кнопка disabled пока поле пустое или чекбокс не отмечен.
- Валидация: non-empty, ≥3 символов.
- Honeypot: скрытое поле `ignore_me` — заполнено → submit игнорируется.
- Обе ссылки (`/privacy`, `/consent`) — **плейсхолдеры `#`** на старте, подставляются перед запуском.

### 6.4. Блок «Что получите в ответ»

Под формой, компактно:
- Точный расчёт потерь на ваших цифрах
- 2–3 первых шага, которые можно внедрить за неделю
- Бесплатно, без обязательств

### 6.5. Экран-благодарность после fallback-формы

- **Текст:** «Принято. Напишу в Telegram/MAX в течение 24 часов»
- **Под текстом, одной строкой:** «Пока ждёте — [посмотрите, как это работает у других]» → ссылка на TG-канал фрилансера или первый кейс. Удержание без давления.

### 6.6. Exit-пути для не-ЦА

**Клиники (`out_clinic`):**

```
Для клиник у меня отдельный формат —
не квиз, а разбор на ваших цифрах.

Оставьте @username, напишу лично.

[ та же форма: @username + чекбокс + отправить ]
```

**Другое (`out_other`):**

```
Сейчас фокусируюсь на нишах «эксперты-одиночки»
и «салоны красоты». Но если ваша ситуация близко —
оставьте контакт, я скажу честно, подходим мы или нет.

[ та же форма ]
```

После сабмита — тот же экран-благодарности, но без ссылки на кейс (у них другой JTBD).

## 7. Тех-стек и структура проекта

**Стек:** HTML + Vanilla JS (лендинг + квиз — единый проект) + вайбкодинг-бэкенд (Claude Code, Node.js/Deno) + TG-бот + Яндекс Таблицы через API + Timeweb (хостинг).

**Почему без фреймворка:** Next.js / Astro / Tilda избыточны для статического квиза из 7 экранов. Вайбкодинг идёт на чистом HTML/JS.

**Почему Яндекс Таблицы, а не Google Sheets:**
- 152-ФЗ: данные под российской юрисдикцией.
- Нативно интегрируется с вайбкодинг-стеком через прямые HTTP-запросы к API.
- До 500 лидов справится. Миграция на Postgres/SQLite — при >300 лидов/мес или при потребности в поиске/фильтрации.

### 7.1. Структура файлов

```
/
├── index.html                    — лендинг (без квиза)
├── quiz.html                     — автономная страница квиза (standalone, для тестов)
├── /assets
│   ├── /css
│   │   └── quiz.css              — все стили с префиксом .quiz-*
│   ├── /js
│   │   ├── quiz-embed.js         — единая точка входа, подключается в лендинг
│   │   ├── quiz.js               — state-машина, навигация, рендер
│   │   ├── calculator.js         — формулы потерь
│   │   ├── scoring.js            — A/B/C/X-тег
│   │   ├── webhook.js            — POST в бэк
│   │   └── analytics.js          — события Метрики
│   └── /img
├── /privacy/index.html           — Политика конфиденциальности (плейсхолдер)
├── /consent/index.html           — Согласие на ОПД (плейсхолдер)
└── /robots.txt, /sitemap.xml
```

### 7.2. Встраивание квиза в лендинг

Лендинг и квиз разрабатываются и деплоятся как два артефакта в одном проекте, но правятся независимо.

- В `index.html` лендинга добавляется `<div id="quiz-root"></div>` там, где должен появиться квиз.
- Подключается `<script type="module" src="/assets/js/quiz-embed.js"></script>`.
- `quiz-embed.js` инициализирует DOM внутри `#quiz-root`, подгружает CSS и модули квиза.
- Все CSS-правила квиза с префиксом `.quiz-` → изолированы от стилей лендинга без iframe.
- Автономная страница `/quiz.html` нужна для тестов и прямых ссылок (например, из DM-рассылки).

## 8. Data flow и backend

### 8.1. Payload на вебхук

```json
{
  "session_id": "q4h7k2m9p0",
  "ts": "2026-04-24T12:34:56Z",
  "path": "main" | "fallback" | "out_clinic" | "out_other",
  "answers": {
    "q1": "pain_missed_records",
    "q2": "format_salon",
    "q3": ["yclients", "tg_built_in_bot"],
    "q4a": "price_3_6k",
    "q4b": "missed_4_7",
    "q5": "urgency_now"
  },
  "calc": {
    "scenario": "salon_missed",
    "loss_min": 47000,
    "loss_max": 75000
  },
  "scoring": {
    "tag": "A",
    "signals": {
      "urgency": "now",
      "scale": "mid",
      "tools": ["yclients"]
    }
  },
  "contact": {
    "username": "@ivan_boss" | null
  },
  "consent": true | false,
  "meta": {
    "utm_source": "tg_chat_beauty",
    "utm_medium": "dm",
    "utm_campaign": "cold_01",
    "referrer": "https://t.me/...",
    "user_agent": "Mozilla/...",
    "locale": "ru-RU",
    "viewport": 390,
    "time_per_q": [8, 3, 12, 5, 4]
  }
}
```

**Инварианты:**
- `consent = true` обязателен для `path in {fallback, out_clinic, out_other}`.
- В `main`-пути `contact.username = null` на момент POST (контакт привязывается ботом по session_id).
- `viewport` — ширина экрана в px, для аналитики мобильного трафика.
- `time_per_q` — секунды на каждый вопрос; диагностика UX (если Q4 > 30 сек — вопрос слишком сложный).

### 8.2. Backend — логика на вайбкодинге

**Компоненты:**

1. **Webhook-endpoint** (Node.js / Deno на Timeweb) — принимает POST с лендинга.
2. **Модуль записи в Яндекс Таблицы** — прямые HTTP-запросы к API.
3. **Модуль TG-бота** (grammY / telegraf) — обрабатывает `/start sess_xxx` и admin-уведомления.
4. **Cron-job** — раз в 15 минут проверяет «повисшие» `sess_xxx` старше 2 часов без привязанного @username.

**Структура листа Яндекс Таблиц** (с запасом под все поля payload):

| колонка | тип | описание |
|---------|-----|----------|
| session_id | string (PK) | ключ лида |
| ts | datetime | время сдачи квиза |
| path | enum | main / fallback / out_clinic / out_other |
| niche | enum | expert / salon / clinic / other |
| scenario | string | expert_commission / expert_flow / expert_rutina / salon_missed |
| loss_min, loss_max | int | диапазон потерь |
| tag | enum | A / B / C / X |
| urgency | enum | now / month / quarter / looking |
| scale_signal | enum | low / mid / high |
| tools | string (csv) | `yclients,tg_built_in_bot` |
| q1…q5 | string/csv | сырые ответы |
| contact_username | string | @username, заполняется ботом |
| contact_attached_at | datetime | когда бот привязал @username |
| consent | bool | галочка пользователя |
| viewport | int | ширина экрана |
| time_per_q | string (csv) | `8,3,12,5,4` |
| utm_source, utm_medium, utm_campaign | string | метки трафика |
| referrer, user_agent, locale | string | meta |
| notified_owner_at | datetime | когда вам пришло уведомление |
| hanging_reminder_sent | bool | флаг cron'а — напоминание «повисшего» |

### 8.3. Путь лида — шаг за шагом

**Main path (TG deep link):**

1. Фронт считает потери, показывает финал.
2. Клик «Получить в TG» → фронт POST'ит payload на вебхук (await ≤3 сек).
3. Бэк записывает строку в Яндекс Таблицы (session_id как ключ), шлёт вам в личный чат уведомление:
   > «Новый лид. Тег: A. Ниша: Салон. Потери: 47–75к ₽. Срочность: прямо сейчас. Инструменты: YCLIENTS. Ожидает в боте — sess_xxx.»
4. Фронт открывает `t.me/YourBot?start=sess_xxx` (даже если вебхук вернул >3 сек — всё равно открывает, чтобы не потерять лид; событие `quiz_webhook_error`).
5. Пользователь жмёт /start в боте → бот находит `sess_xxx`, привязывает `@username`, отвечает:
   > «Принято, @ivan_boss. Готовлю расчёт на ваших цифрах, отпишу лично в течение 24 часов. Пока можете задать любой вопрос прямо здесь.»
6. Вам приходит апдейт:
   > «@ivan_boss привязан к sess_xxx. Можно писать.»

**Fallback path (форма):**

1. Фронт валидирует форму.
2. POST на вебхук (await response, spinner).
3. Бэк записывает, шлёт вам уведомление с `@username`.
4. Фронт: экран-благодарности + ссылка на TG-канал.

**Exit paths (`out_clinic`, `out_other`):**

1. Та же форма.
2. POST с флагом path.
3. Бэк шлёт вам уведомление с меткой out-типа.
4. Фронт: тот же экран-благодарности.

**Cron — «повисшие» sess_xxx:**

Раз в 15 минут. Критерий:
- `ts` старше 2 часов.
- `path = "main"`.
- `contact_username = null`.
- `hanging_reminder_sent = false`.

На совпадение — одно уведомление вам:
> «Лид sess_xxx зашёл в бота 2ч назад, не нажал /start. Тег A. Потери 47–75к. Ниша: Салон. Решите — писать в общий канал или нет.»

Флаг `hanging_reminder_sent = true` — больше не трогаем этот sess_xxx.

| Событие | Параметры | Для чего |

## 10. Обработка ошибок и edge-cases

| Сценарий | Поведение |
|---------|-----------|
| Webhook timeout >3 сек на main-пути | Открываем deep link всё равно, повторяем POST в фоне до 3 попыток. Событие `quiz_webhook_error`. |
| Webhook fail на fallback-форме | Красная строка «Что-то пошло не так, попробуйте ещё раз». Retry-кнопка. |
| sessionStorage заблокирован | Работаем на in-memory state. Refresh = потеря прогресса. |
| Refresh посреди квиза | Восстанавливаем из sessionStorage, если есть; иначе — назад на hero. |
| Abandon на финале без клика | Только событие в Метрике `abandon_at_final`. Payload не отправляем. |
| Double-click на главной кнопке | Debounce 1 сек + disabled после первого клика. |
| Spam в fallback-форме | Honeypot-поле `ignore_me` + rate limit 5 POST/IP/мин на вебхуке. |
| JS отключен | `<noscript>`: «Для квиза нужен JavaScript. Или напишите напрямую: [deep link к боту]». |
| User без Telegram | Fallback-форма решает. |
| User с TG, но не жмёт /start после deep link | Через 2 часа — одно уведомление вам через cron (см. 8.3). |

## 11. Мобильная адаптация и доступность

- Mobile-first: основной дизайн 320–768 px.
- Шрифты ≥ 16 px (iOS не зумит). Touch-target кнопок ≥ 52 px.
- Hero на mobile — 100vh. Прогресс-бар sticky-top.
- Safe-area insets для iPhone с чёлкой/home-indicator.
- Нативные `<button>`, `<input>`, `<label>`, `<a>` — никаких `<div role="button">`.
- Контраст текст/фон ≥ 4.5:1 (WCAG AA). Видимый focus-outline.
- Keyboard navigation работает (Tab через все интерактивные).
- ARIA-label на иконочных кнопках (назад).

## 12. Что НЕ входит в MVP

- Полноценная БД (Postgres/SQLite) — Яндекс Таблицы хватит до 300–500 лидов.
- A/B-тестирование вариантов вопросов — только после набора первых 100 лидов.
- Многошаговые персонализированные email-воронки — у фрилансера нет email-стека, вся коммуникация в TG.
- Локализация на английский/СНГ-языки — целевой рынок РФ + Казахстан/Беларусь (русскоязычные).
- Отдельные лендинги под ниши — один лендинг, квиз ветвится внутри.
- Визуальные диаграммы потерь (bar chart, donut) — горизонтальная полоса достаточна, остальное — карго-культ на мобильном.
- Интеграция с CRM (Bitrix24/amoCRM) — вы фрилансер-одиночка, CRM не нужна.
- Премиум-трек (X-тег, пакет 150–300к) — скоринг X помечает лида, но отдельного сценария на лендинге нет; обрабатывается вручную.
